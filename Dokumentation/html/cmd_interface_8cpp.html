Detailed Description

Implementierung des seriellen Kommando-Interfaces und der Parameterverwaltung.

  • Dieses Modul verarbeitet die eingehenden seriellen ASCII-Befehle asynchron, steuert die zentrale Zustandsmaschine (FSM) des Systems und bietet Funktionen zum softwareseitigen Mapping sowie zur persistenten Speicherung der Sensor-Kalibrierdaten im Flash-Speicher.

◆ handleSerialInput()

void handleSerialInput ( )

Liest den Hardware-UART-Puffer zeilenweise aus.

Überwacht und liest den seriellen Eingangspuffer aus.

  • Die Funktion liest nicht-blockierend alle verfügbaren Bytes. Bei Erkennung eines Zeilenumbruchs (CR/LF) wird der String nullterminiert, in den Arbeitspuffer cmdBuf kopiert und das Verarbeitungsflag cmdReady gesetzt. Bei Pufferüberlauf wird der bisherige Inhalt verworfen (Überlaufschutz).

Liest den Hardware-UART-Puffer zeilenweise aus.

  • Diese Funktion muss zyklisch in der Hauptschleife (loop) aufgerufen werden. Sie puffert eingehende ASCII-Zeichen, bis ein vollständiger Befehl vorliegt.

◆ processCommand()

void processCommand ( )

Zentrale State-Machine und Befehls-Interpreter.

Interpretiert und führt das empfangene serielle Kommando aus.

  • Falls cmdReady gesetzt ist, wird das Flag sofort zurückgesetzt und das erste Zeichen des Puffers extrahiert. Über ein switch-case-Konstrukt werden entweder kontinuierliche Sensor-Samplings (A, G, M) gestartet, das System gestoppt (S), Parameter modifiziert/gelesen (s, r) oder die RAM-Struktur prefs persistent in den Flash-Speicher gesichert (w).

Zentrale State-Machine und Befehls-Interpreter.

  • Analysiert das Befehlszeichen (z.B. 'A', 'G', 'M', 'S', 's', 'r', 'w') und stößt den entsprechenden Zustandswechsel oder Speicherzugriff an.

Parser- und Setzfunktion für das serielle Kommando 's'.
